# 武學招式進化系統 Spec

## 1. 目標

讓「武學招式」與「對應心法」形成明確組合。當玩家同時修練指定招式與心法到一定層數後，可以在升級時領悟「武林絕學」。

本系統同時加入「孤本心法」：這類心法不能與招式結合成絕學，但單體效果比一般心法更強，讓玩家在湊配方與拿強力泛用被動之間做取捨。

系統目的：

- 讓升級選擇不只是數值堆疊，而是有明確武俠流派與終局目標。
- 讓 Boss 解鎖心法後，玩家有新的追求與 Build 路線。
- 讓每種主力招式都有進化方向，提升中後期爽感。
- 讓玩家能從「招式 + 心法 + 流派」形成可辨識的武學風格。
- 讓孤本心法提供高價值但不能進化的替代選擇。

## 2. 設計原則

- 武林絕學應該是原招式的強化與變體，而不是完全無關的新技能。
- 進化條件需要清楚，玩家要知道自己正在湊哪個組合。
- 進化後應保留原招式定位，但加入一個明顯的新能力。
- 每場遊戲可以進化多個招式，但每個招式只進化一次。
- UI 要把進化選項視覺上和普通升級區分開。
- 孤本心法不參與任何進化配方，但其數值強度要高於一般心法。

## 3. 進化條件

一個招式可以進化成武林絕學，需要同時符合：

- 指定招式達到 Lv.5。
- 指定心法達到指定層數，預設 Lv.3。
- 該心法已被 Boss 階段解鎖。
- 該心法不是孤本心法。
- 該招式尚未進化。
- 玩家升級時，進化選項會加入升級池。

建議規則：

| 條件 | 預設值 |
|---|---|
| 招式等級 | Lv.5 |
| 心法等級 | Lv.3 |
| 進化時機 | 升級選項中出現 |
| 每次升級進化選項上限 | 1 個 |
| 同一招式可進化次數 | 1 次 |
| 孤本心法可否進化 | 不可 |

## 4. 十組招式進化配方

命名方向：

- 招式名稱偏向「劍訣、掌法、身法、陣勢」，避免太像功能描述。
- 心法名稱偏向「內功、神功、訣、篇」，讓玩家感覺是在修練秘笈。
- 武林絕學名稱要像流派終局招牌，一看就知道是進化後的大成境界。

### 4.1 華山劍氣 + 獨孤劍意 = 獨孤破式劍

| 項目 | 內容 |

| 原招式 | 華山劍氣 / Sword Qi |
| 對應心法 | 獨孤劍意 |
| 武林絕學 | 獨孤破式劍 / Void-Cleaving Dugu Sword |
| 定位 | 高傷害、穿透、暴擊 |

效果規劃：

- 華山劍氣變成寬幅劍芒。
- 穿透數大幅提升。
- 暴擊時產生小範圍裂空傷害。
- 劍宗流等級越高，裂空傷害越強。

## 5. 孤本心法

孤本心法是高價值被動心法，不能與任何招式結合，也不會觸發武林絕學。它的定位是「少一條進化路線，換取更強泛用能力」。

### 5.1 共通規則

| 規則 | 建議值 |
|---|---|
| 是否可進化 | 不可 |
| 最高等級 | Lv.3 |
| 出現權重 | 一般心法的 40% 到 60% |
| 每局取得上限 | 建議 1 到 2 本 |
| 解鎖來源 | Boss 階段、聲望、成就或中後期升級池 |
| 數值強度 | 每級約為一般心法的 1.25 到 1.5 倍 |

### 5.2 第一批孤本心法

| 孤本心法 | 定位 | 效果規劃 |
|---|---|---|
| 易筋鍛骨篇 / Tendon-Bone Forging Manual | 生存、減傷、最大氣血 | 提升最大氣血與受傷後短暫減傷，Lv.3 可增加瀕死保護冷卻縮短 |
| 乾坤吐納訣 / Cosmos Breathing Method | 冷卻、內力恢復 | 提升冷卻縮減與內力恢復，滿級時升級後短暫加快所有招式循環 |
| 無相心經 / Formless Heart Sutra | 泛用增傷、範圍 | 小幅提升所有傷害與招式範圍，不吃特定流派但數值穩定 |
| 洗髓經 / Marrow-Cleansing Classic | 回復、容錯 | 提升回血效率，低血量時提高經驗吸取與撿拾半徑 |
| 逍遙御風訣 / Freewind Wandering Method | 身法、閃避、拾取 | 提升移速、閃避與拾取半徑，滿級時閃避成功會短暫提高經驗吸取 |

### 5.3 平衡限制

- 孤本心法不得出現在 `EvolutionConfig.requiredSkillId`。
- 孤本心法的升級卡片要標示「孤本心法」。
- 孤本心法不應阻塞武林絕學選項，但可以與普通心法競爭升級池位置。
- 若一局已取得 2 本孤本心法，後續孤本權重歸零。
- Dev/Test mode 需要可以強制抽到孤本心法，方便驗證數值與 UI。

## 6. 解鎖與顯示規則

### 6.1 升級池規則

當玩家升級時：

1. 先檢查所有進化配方。
2. 找出符合條件且尚未進化的配方。
3. 過濾掉任何需要孤本心法的錯誤配方。
4. 若存在可進化配方，最多放入 1 個進化選項。
5. 進化選項應優先於普通武器升級，但不必每次都必定出現。

建議權重：

- 可進化選項出現率：70%
- 若同時有多個可進化配方，隨機選 1 個
- 孤本心法出現率：一般心法的 40% 到 60%
- Dev/Test mode 可提供 debug 方式強制檢查進化池與孤本池

### 6.2 UI 表現

進化選項要和普通升級不同：

- 標題使用「領悟武林絕學」。
- 卡片邊框使用金色或特殊光效。
- 顯示配方：
  - 已修成招式
  - 已修成心法
  - 進化後名稱
- 選擇後顯示畫面提示：
  - `領悟武林絕學：獨孤破式劍`

孤本心法卡片：

- 標題使用「孤本心法」。
- 使用不同於進化卡片的視覺標記，避免玩家誤以為能進化。
- 描述要明確標示「不可結合招式」。

### 6.3 HUD 表現

HUD 中的招式列表要顯示進化後名稱。

例如：

```text
招式
獨孤破式劍 第 1 重
吸星歸元大法 第 1 重

心法
易筋鍛骨篇 第 2 重
```

## 7. 資料結構規劃

### 7.1 新增資料檔

建議新增：

```text
src/data/evolutions.ts
```

資料型別：

```ts
export type EvolutionId =
  | "voidDuguSword"
  | "windClearSwordArray"
  | "starDrainingPalm"
  | "drunkenShadowNineSwords"
  | "starReturningOriginField"
  | "violetMistBlossomSword"
  | "shadowlessGaleSlash"
  | "taiyueMountainSealingForm"
  | "coldPondMirrorSword"
  | "vajraHundredStepQuake";

export type EvolutionConfig = {
  id: EvolutionId;
  nameKey: string;
  baseWeaponId: WeaponId;
  requiredWeaponLevel: number;
  requiredSkillId: SkillId;
  requiredSkillLevel: number;
  preferredBuildPathId?: BuildPathId;
  descriptionKey: string;
};
```

### 7.2 SkillConfig 擴充

建議在心法資料加入分類：

```ts
export type SkillKind = "combo" | "standalone";

export type SkillConfig = {
  id: SkillId;
  nameKey: string;
  descriptionKey: string;
  kind: SkillKind;
  maxLevel: number;
  tags?: Array<"sword" | "qi" | "movement" | "survival" | "rare">;
};
```

規則：

- `combo` 心法可出現在進化配方中。
- `standalone` 心法就是孤本心法，不可出現在進化配方中。
- 測試需檢查所有 `EvolutionConfig.requiredSkillId` 都不是 `standalone`。

### 7.3 GameState 擴充

建議新增：

```ts
evolvedWeapons: Map<WeaponId, EvolutionId>;
standaloneSkillLevels: Map<SkillId, number>;
```

用途：

- 判斷某個招式是否已進化。
- HUD 顯示進化後名稱。
- WeaponSystem 決定使用原招式邏輯或進化邏輯。
- 升級池判斷孤本心法取得數量與等級。

### 7.4 UpgradeOption 擴充

可選擇加入 `kind`，方便 UI 區分卡片：

```ts
kind?: "stat" | "weapon" | "skill" | "build" | "evolution" | "standaloneSkill";
```

若暫時不想改動太大，也可以用 `id` 前綴 `evolution-` 與 `standalone-skill-` 判斷。

## 8. WeaponSystem 行為規劃

每個原招式保留原方法，內部依進化狀態分流。

建議方式：

```ts
const evolution = evolvedWeapons.get(weaponId);
if (evolution === "voidDuguSword") {
  this.fireVoidDuguSword(level);
  return;
}
```

第一階段可先做數值與視覺上的強化，不必一次做很複雜。

建議 MVP：

| 武林絕學 | MVP 先做 |
|---|---|
| 獨孤破式劍 | 更大劍氣、更高穿透、暴擊裂空 |
| 清風十三劍陣 | 更多劍輪、半徑提升 |
| 吸星摧心掌 | 氣浪吸血與輕微牽引 |
| 醉裡破式九劍 | 多目標與追加斬 |
| 吸星歸元大法 | 力場持續更久、回血增加 |
| 紫霞落英神劍 | 扇形散射與回旋二段傷害 |
| 狂風無影劍 | 移動蓄勢與追加斬擊 |
| 太岳鎮嶽劍 | 重擊落點、擊退、破防 |
| 寒潭照影十三劍 | 寒氣區域與殘影劍痕 |
| 金剛百步伏魔拳 | 直線震波與短暫護體 |

## 11. 測試規劃

新增 Vitest 測試：

- 未達條件時，升級池不出現進化選項。
- 招式 Lv.5 + 心法 Lv.3 時，升級池可出現進化選項。
- 選擇進化後，`evolvedWeapons` 會記錄對應配方。
- 已進化招式不會再次出現同一進化選項。
- 10 組進化配方都有合法招式與心法。
- 孤本心法不會出現在任何進化配方。
- 孤本心法不會產生進化選項。
- 孤本心法最高等級與每局取得上限符合規則。
- 進化後 HUD 名稱可從 i18n 找到。
- 繁中/英文 evolution 與 standalone key 不缺漏。
